## Порядок запуску (усе під схемою BAS_REVERSE, NLS_LANG=.AL32UTF8)

1. `raw-ddl.sql` — RSD_ODATA_RAW (no-op, вже існує з батча 004).
2. `stage.sql` — вибірка кожного `Catalog_*` у RAW (**$skip-пагінація**).
3. `relax-constraints.sql` — знімає UNIQUE(CODE) на 3 довідниках (seed↔live конфлікт Code).
4. `project.sql` — типізований MERGE RAW→RSD_* (двічі, реконсиляція циклів).
5. `dedup-seed.sql` — прибирає seed-залишки (LEGACY_REF немає в RAW).
6. `reconcile-refs.sql` — крос-FK за field-map (напр. Руководитель→RSD_POLZOVATELI 153/155).
7. `reconcile-parent.sql` — self-FK ієрархії (напр. СтруктураПідприємства 144/155).

## Рішення та пастки цього батча

- **$skip замість keyset Ref_Key.** `gen_odata_loaders` мав keyset-пагінацію за
  `Ref_Key gt`, яка тихо обривала сутності >500 на першій сторінці (1С ігнорує
  `$orderby`). Виправлено на `$skip` (надійно ≤~3000). >3000 — окремий Code-prefix.
- **relax UNIQUE(CODE).** Seed (предопределённые з батча 003, LEGACY_REF=metadata-UUID)
  і живі записи ділять Code за різними UUID → ORA-00001. Знято DB-UNIQUE (унікальність — рівня APEX).
- **dedup seed.** Предопределённые seed мають metadata-UUID ≠ OData Ref_Key → дублювали
  живі елементи. Прибрано 9 рядків (NUMERATORY −4, +5×−1), лічильники зійшлися з джерелом.
- **reconcile PARENT_ID.** `gen_catalog_batch` не кладе колонку ієрархії у field-map,
  тож загрузчики її не резолвлять — окремий self-FK прохід (**gap генератора, TODO**).
