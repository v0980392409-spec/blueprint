# Батч 007 — §9 рішення власника: OWNER_TYPE + типізовані VALUE_*

Реалізує рішення власника з [`docs/section9-decisions.md`](../../docs/section9-decisions.md)
(2026-07-19). Групи 1 і 3 — підтверджено дефолт (0 змін); групи 2 і 4a — тут.

## Група 2 — поліморфний власник (`OWNER_TYPE`)

`ddl.sql` додає `OWNER_TYPE VARCHAR2(60)` до `RSD_BANKOVSKIESCHETA` і
`RSD_NASTROI_KIRABOCHIKHMESTPOLZOVATELEI` (поряд з `OWNER_ID` — поліморфна пара).

`backfill.sql` (Банк.рахунки, 5752 живих рядки):
- `OWNER_TYPE` ← OData `Owner_Type` → **Контрагенти 4854 / Організації 898**.
- `OWNER_ID` переразрешено **за типом** проти RSD_KONTRAGENTY / RSD_ORGANIZATIONS —
  **5678/5752 (98.7%)** заповнено (74 null — порожній Owner/ціль поза завантаженням).
- **Пастка:** ключ поліморфного посилання 1С OData — у полі **`Owner`** (не
  `Owner_Key`; `<Поле>_Key` лише для однотипних). До цього OWNER_ID був увесь NULL.

## Група 4a — значення доп.реквізитів (типізовані `VALUE_*`)

`ddl.sql` додає до 9 ТЧ `*_DOPOLNITELNYEREKVIZITY` (порожні) замість дропнутого
нетипізованого `Значение`: `VALUE_TYPE` (дискримінатор) + `VALUE_STRING/NUMBER/
DATE/BOOL` + `VALUE_REF_TYPE/VALUE_REF_ID`. Тип значення задає властивість
(`SVOYSTVO_ID`), патерн 1С `ДополнительныеРеквизитыИСведения`. 63 колонки.

## Стан після батча

- `RSD_BANKOVSKIESCHETA`: OWNER_TYPE 5752, OWNER_ID 5678. 0 INVALID на схемі.
- 9 ТЧ доп.реквізитів мають типізовані VALUE_* (порожні — заповняться при завантаженні ТЧ).

## Follow-up (не в цьому батчі)

- **Група 4b — нетипізовані посилання (~19).** Пункти §9 «нетипізований
  (характеристика)» неоднорідні: крім `Значение` там untyped-ПОСИЛАННЯ
  (`ОбъектКТ`, `Подразделение`, `ФизическоеЛицо`, `Организация`, `Источник`…).
  Частина — одноцільові FK, хибно визначені як untyped (напр. `Подразделение`→
  СтруктураПідприємства); частина дублює наявні колонки (`ФизическоеЛицо` vs
  `FIZLITSO_ID`); частина справді поліморфна. Потрібен по-атрибутний розбір
  (одиночний FK де ціль однозначна, інакше REF_TYPE/REF_ID) — окремий міні-батч.
- **Gap генераторів.** `gen_catalog_batch` не кладе у field-map ні PARENT_ID
  (ієрархія), ні поліморфний OWNER (тип+ключ) → загрузчики їх не резолвлять.
  Дані добито окремими проходами (batch 006 reconcile-parent, batch 007 backfill),
  але сам генератор варто виправити для майбутніх батчів.
