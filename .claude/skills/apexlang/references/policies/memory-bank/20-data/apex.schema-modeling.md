# Schema Modeling — Deterministic Conventions (Plan‑Only, Oracle 23.9+/23c/23ai Aware)

Purpose
- Authoritative rules for generating schema DDL plans (no apply) with Oracle best practices.
- Enforce deterministic naming, constraints, audit/row_version patterns, and feature gating by DB version.
- Used by the Schema Modeling constitutional workflow to draft → critique → revise DDL scripts.

Scope and Non‑negotiables
- Plan‑only: never execute DDL; emit scripts and reviews; validation is parse‑only.
- First agent must establish DB connection and record version/capabilities; features are gated accordingly.
- Identifiers: unquoted snake_case, <= 30 chars, allowed [A‑Z0‑9_], avoid reserved words, ensure global uniqueness.
- Object ordering in emission: tables → indexes (incl. FK indexes) → sequences → triggers → comments.
- Statement terminator: every statement ends with “/” on its own line. Triggers end with “;” within body and “/” after.
- All SQL in outputs must be in a single triple‑backticked ```sql code block where required by the workflow.

Naming Conventions
- Tables: <prefix>_<entity>
- Constraints:
  - PK: <prefix>_<entity>_pk
  - UK: <prefix>_<entity>_uk[_n]
  - FK: <prefix>_<entity>_fk[_n]
  - CK: <prefix>_<entity>_ck[_n]
- Indexes: <entity>_indx_n (n = 1..N), never duplicate names.
- Triggers: <prefix>_<entity>_biu (before insert or update)

Primary Keys (surrogate; NUMBER)
- Preferred (>= 23c/23ai/23.9+): GENERATED BY DEFAULT AS IDENTITY (no sequence/trigger for PK).
- Fallback (< 23c): sequence + before insert trigger.
- PK column name: id NUMBER not null; PK constraint named per convention.

Audit Columns and Row Version
- Columns on every table (unless explicitly disabled):
  - created_on DATE not null
  - created_by VARCHAR2(255) not null
  - updated_on DATE not null
  - updated_by VARCHAR2(255) not null
  - row_version INTEGER not null
- BIU trigger responsibilities:
  - On insert: set created_* and updated_*, initialize row_version = 1
  - On update: increment row_version, update updated_* only
  - Determine user with SYS_CONTEXT('APEX$SESSION','APP_USER') fallback USER

Foreign Keys and Indexes
- Always create an index for each foreign key column (or composite columns).
- ON DELETE CASCADE only when explicitly requested; default is ON DELETE NO ACTION.
- FK columns must be NOT NULL unless a nullable relationship is explicitly stated.

Status/Lookup Strategy
- Prefer lookup tables for status/domain values to ensure consistency and extensibility.
- Use a foreign key to the lookup table; avoid hard‑coded CK unless trivial and stable.

JSON and BOOLEAN (Feature Gating)
- JSON:
  - >= 23c/23ai: use JSON data type columns; optionally add JSON validity constraints when beneficial.
  - < 23c: use CLOB with IS JSON check constraint.
- BOOLEAN:
  - >= 23c: BOOLEAN type permitted.
  - < 23c: emulate with CHAR(1) and CK IN ('Y','N') or a lookup table.

Temporal Types
- Default to DATE for general timestamps; use TIMESTAMP WITH LOCAL TIME ZONE when precision/time zone required.
- Avoid TIMESTAMP WITHOUT TZ for cross‑time‑zone data unless justified.

Comments
- Emit COMMENT ON TABLE and COMMENT ON COLUMN for every object to aid data dictionary readability.

Privileges and Security
- Do not emit GRANT/REVOKE or ACL DDL in modeling outputs; document prerequisites in reporting.
- Never disable RLS/VPD; designs must work within existing policies.

Formatting and Emission Rules
- One object per statement block; end each with “/”.
- Place constraints inline in CREATE TABLE where practical; use ALTER TABLE ADD CONSTRAINT when needed.
- Generate FK indexes explicitly (CREATE INDEX …) after tables.
- For pre‑23c fallback, place sequences and triggers after indexes.

Examples (snippets; use version gating in the workflow to choose path)

-- Identity PK (>= 23c)
```sql
create table inv_item (
    id                            number generated by default as identity
                                                 constraint inv_item_pk primary key
  , name                          varchar2(255)                                 
  , description                   varchar2(4000)                                
  , row_version                   integer        not null                        
  , created_on                    date           not null                        
  , created_by                    varchar2(255)  not null                        
  , updated_on                    date           not null                        
  , updated_by                    varchar2(255)  not null                        
)
/ 

comment on table inv_item is 'Inventory: items';
/
comment on column inv_item.name is 'Item display name';
/
```

-- Sequence + Trigger PK (< 23c)
```sql
create table inv_supplier (
    id                            number         not null
                                                 constraint inv_supplier_pk primary key
  , name                          varchar2(255)                                 
  , row_version                   integer        not null                        
  , created_on                    date           not null                        
  , created_by                    varchar2(255)  not null                        
  , updated_on                    date           not null                        
  , updated_by                    varchar2(255)  not null                        
)
/
create sequence inv_supplier_seq
/
create or replace trigger inv_supplier_biu
  before insert or update on inv_supplier
  for each row
begin
  :new.updated_on := sysdate;
  :new.updated_by := coalesce(sys_context('APEX$SESSION','APP_USER'), user);
  if inserting then
    if :new.id is null then
      select inv_supplier_seq.nextval into :new.id from dual;
    end if;
    :new.created_on := :new.updated_on;
    :new.created_by := :new.updated_by;
    :new.row_version := 1;
  else
    :new.row_version := nvl(:old.row_version, 0) + 1;
  end if;
end inv_supplier_biu;
/
```

-- JSON and BOOLEAN (>= 23c) with fallback comments
```sql
create table hr_employee (
    id                            number generated by default as identity
                                                 constraint hr_employee_pk primary key
  , active                        boolean                               -- <23c: use char(1) with CK or lookup
  , profile                       json                                  -- <23c: use clob with IS JSON
  , row_version                   integer        not null
  , created_on                    date           not null
  , created_by                    varchar2(255)  not null
  , updated_on                    date           not null
  , updated_by                    varchar2(255)  not null
)
/
```

-- Foreign key and index
```sql
alter table inv_item
  add constraint inv_item_supplier_fk
  foreign key ( supplier_id )
  references inv_supplier ( id )
/
create index inv_item_indx_1 on inv_item ( supplier_id )
/
```

Validation (Parse‑Only) Guidance
- Use SQLcl with `db_connection_name` to establish the validation session.
- Validate statements with parse‑only techniques; never execute DDL.
- Record diagnostics with line references into checklist/reporting.

Gates and Violations
- Hard‑fail:
  - Missing/invalid db_connection_name for this domain.
  - Identifier length/charset violations; reserved keywords used as identifiers.
- Defer with review:
  - ON DELETE CASCADE not explicitly justified.
  - Unindexed foreign keys.
  - Non‑deterministic or non‑portable features without version gating.

Compatibility with Other Rules
- Complements 20-data/apex.sql.md (UI query rules). Keep DDL logic separate from UI SQL.
- Follows repository governance and output policy (APEXlang domain; SQL in ```sql blocks; no UT CSS or APEX attributes here).
