# utPLSQL Boundary Rules (APEXlang Local Policy)

## Purpose

- Treat Oracle upstream DB skills at `https://github.com/oracle/skills/tree/main/db` as the main source of truth for generic utPLSQL guidance.
- Keep this file limited to APEXlang-local boundaries for generated APEX artifacts and repository workflow safety.
- Avoid vendoring, restating, or replacing upstream utPLSQL authoring, reporter, coverage, installation, or CI guidance.

## Scope

- Applies only when APEXlang or the local PL/SQL integration adapter must reason about tests for PL/SQL created from, extracted from, or used by APEX artifacts.
- Does not make the APEXlang package a standalone utPLSQL skill or generic PL/SQL testing authority.
- Does not authorize live test execution without the normal DB metadata, SQLcl connection, and runtime safety gates.

## Local Boundary Rules

- Keep utPLSQL source authoring separate from utPLSQL execution eligibility.
- Allow test planning or test artifact authoring without a UT3 runtime in the target schema.
- Require a UT3 preflight in the target schema before any utPLSQL run plan is treated as executable.
- If UT3 is absent, mark execution as `blocked`; continue only with planning, package compilation checks, direct smoke checks, or reporting outputs that do not require UT3.
- Do not frame the next step as installing utPLSQL from this package; direct generic setup guidance to upstream Oracle DB skills.
- Never vendor utPLSQL sources into this repository or the public APEXlang package.

## APEXlang-Specific Guardrails

- When `app_process_api` exists or is generated by a maintenance flow, default test scope must include it.
- Block test plans that target page-number-based package names such as `p123_*`; package names should be neutral and domain-oriented.
- Keep test artifacts and reports under the repository artifact boundary:
  - compact run logs under ``APEXLANG_OUTPUT_ROOT/logs/utplsql/``
  - durable reports under ``APEXLANG_OUTPUT_ROOT/reports/utplsql/``
- Redact credentials, full connect strings, and workspace-sensitive details from any test report or log.

## Local Workflow Boundary

- Local PL/SQL integration adapter testing operation:
  - discovers local test scope,
  - checks execution readiness,
  - records blocked execution when prerequisites are missing,
  - emits local reporting artifacts.
- Local PL/SQL integration adapter maintenance operation:
  - may emit suggested tests for extracted package APIs,
  - does not become a generic utPLSQL design authority.
