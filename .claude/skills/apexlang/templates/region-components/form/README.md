# Form Templates

## Purpose
Canonical guidance for the `form` region family, including shared contract loading and supported scenario variants.

## Usage
- Load `form._index.md` first.
- Load `form._common.md` next to align variable contracts, guardrails, and required inputs.
- Load `form._template_options.md` when the request changes form-region or companion button-region `appearance.templateOptions`.
- Load `form._items._common.md` for canonical form item contracts.
- Load `form._processes._common.md` for canonical process linkage and placeholders.
- Load one scenario variant matching the requested interaction pattern and data source type.
